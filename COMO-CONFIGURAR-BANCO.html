🗄️ Configurar o Banco de Dados

Supabase · gratuito · 5 minutos · dados salvos na nuvem para toda a equipe

1
Criar conta gratuita no Supabase
Sem cartão de crédito · plano gratuito é suficiente
Acesse o site do Supabase e clique em Start your project ou Sign up.
Recomendamos entrar com a conta do GitHub ou Google — mais rápido.
→ supabase.com
2
Criar um novo projeto
Leva ~2 minutos para o banco ficar pronto
No painel, clique em New project.
Preencha:
• Name: financeiro-escola (ou qualquer nome)
• Database Password: crie uma senha forte (anote em local seguro)
• Region: escolha South America (São Paulo)
Clique Create new project e aguarde ~2 minutos enquanto o banco é criado.
3
Criar a tabela no banco
Uma única tabela que armazena todo o estado do sistema
No painel do projeto, clique em SQL Editor no menu esquerdo.
Clique em New query, cole o código abaixo e clique Run (▶):
-- Cria a tabela que armazena todos os dados do sistema CREATE TABLE IF NOT EXISTS financeiro_state ( id TEXT PRIMARY KEY, turmas JSONB, receitas JSONB, categorias JSONB, lancamentos JSONB, rec_lancamentos JSONB, meses JSONB, active_months JSONB, school TEXT, year TEXT, updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Libera acesso (o sistema usa a chave anon para ler e gravar) ALTER TABLE financeiro_state ENABLE ROW LEVEL SECURITY; CREATE POLICY "acesso_total" ON financeiro_state FOR ALL USING (true) WITH CHECK (true); -- Ativa atualizações em tempo real ALTER PUBLICATION supabase_realtime ADD TABLE financeiro_state;
💡 Se aparecer "Success. No rows returned" ou "already exists", está correto — a tabela foi criada.
4
Copiar as credenciais de acesso
Você vai colar esses valores nas Configurações do sistema
No painel do projeto, clique em Settings (ícone de engrenagem) → API.
Copie os dois valores:

📌 Project URL — parece com:
https://abcdefghijk.supabase.co

📌 anon public (em "Project API keys") — começa com eyJhbGci...
⚠️ Use sempre a chave anon (pública), nunca a service_role (privada). A anon já tem as permissões corretas que configuramos.
5
Conectar o sistema ao banco
Cole as credenciais nas Configurações do dashboard
No sistema, vá na aba ⚙️ Configurações.
Cole a Project URL no campo URL do projeto Supabase.
Cole a anon key no campo Chave anon/public.
Clique 🔌 Conectar ao banco.
Se aparecer 🟢 Conectado ao banco · salvamento automático ativo, está funcionando!
💡 As credenciais são salvas no navegador — você só precisa configurar uma vez por dispositivo. Cada pessoa da equipe configura no próprio computador/celular.
✅ Pronto! Como funciona agora:
• Cada lançamento, edição ou exclusão é salvo automaticamente no banco em ~1 segundo
• Se outra pessoa da equipe estiver com o sistema aberto, ela vê a atualização em tempo real
• Os dados ficam salvos mesmo se fechar o navegador, desligar o computador ou trocar de dispositivo
• O plano gratuito do Supabase aguenta 500MB de dados e 50.000 requests/mês — mais do que suficiente para uma escola
