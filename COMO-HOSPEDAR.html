1
Publicar o site no Netlify
~5 minutos · requer criar conta gratuita
1
Criar conta no Netlify
Acesse o link abaixo e clique em Sign up. Recomendamos entrar com a conta do Google — é mais rápido.
→ app.netlify.com/signup
2
Fazer upload dos arquivos
Após entrar, na tela inicial clique em Add new site → Deploy manually.
Arraste a pasta escola-financeiro inteira para a área indicada.
Aguarde o upload (alguns segundos).
📁Arraste a pasta inteira, não os arquivos individualmente. O Netlify precisa ver o index.html na raiz.
3
Renomear o site
Após o upload, você recebe um nome aleatório tipo amazing-cupcake-123.netlify.app.
Para mudar: vá em Site configuration → Change site name.
Escolha algo como financeiro-minha-escola.netlify.app.
✅Ao terminar essa fase, seu site já está acessível publicamente. A próxima fase vai adicionar a proteção por login.
2
Criar conta no Cloudflare e apontar o domínio
~10 minutos · necessário para ativar o bloqueio de acesso
1
Criar conta gratuita no Cloudflare
Acesse o link e clique em Sign up for free. Use o mesmo e-mail que você usa no trabalho.
→ dash.cloudflare.com/sign-up
2
Ativar o Cloudflare Zero Trust
No painel do Cloudflare, no menu esquerdo clique em Zero Trust.
Vai pedir para configurar um nome de equipe — pode colocar o nome da escola, ex: minha-escola.
Escolha o plano Free (gratuito).
→ one.dash.cloudflare.com
⚠️O Cloudflare Access funciona com domínios próprios (ex: minhaescola.com.br) OU com o domínio do Cloudflare Pages. Para o caminho mais simples, vamos usar o Cloudflare Pages direto em vez do Netlify — isso elimina a necessidade de um domínio próprio. Veja a Fase 2B abaixo.
2B
✅ Caminho mais simples: Cloudflare Pages direto
Hospedagem + proteção tudo no Cloudflare · sem precisar de domínio próprio
1
Ir para Cloudflare Pages
No painel do Cloudflare, menu esquerdo: Workers & Pages → Pages → Create a project → Upload assets.
→ dash.cloudflare.com → Pages
2
Fazer upload da pasta
Clique em Upload assets, dê um nome ao projeto (ex: financeiro-escola),
arraste a pasta escola-financeiro e clique Deploy site.
Você recebe um link como financeiro-escola.pages.dev.
3
Criar a política de acesso (o "bloqueio")
Ainda no Cloudflare, vá em Zero Trust → Access → Applications → Add an application.
Escolha Self-hosted.

Preencha:
• Application name: Dashboard Escola
• Application domain: financeiro-escola.pages.dev
• Clique Next.
4
Configurar quem pode entrar
Na tela de políticas:
• Policy name: Equipe Escola
• Action: Allow
• Em Configure rules → Include, selecione Emails
• Digite os e-mails autorizados um por um (ex: diretor@escola.com.br, secretaria@escola.com.br)

Clique Next → Add application.
💡Você pode adicionar ou remover e-mails a qualquer momento voltando nesta tela. Não precisa reenviar nenhum arquivo.
5
Testar o acesso
Abra uma aba anônima (Ctrl+Shift+N) e acesse financeiro-escola.pages.dev.
Deve aparecer uma tela do Cloudflare pedindo o e-mail.
Digite um e-mail autorizado → recebe um código de 6 dígitos → digita o código → entra no sistema.

Quem não tiver o e-mail na lista não consegue nem ver a página.
3
Compartilhar dados entre a equipe
Os dados ficam salvos no navegador de cada usuário — veja como sincronizar
1
Como o salvamento funciona
Os dados ficam salvos automaticamente no navegador de cada pessoa.
Isso significa: se você lançar uma receita no seu computador, sua colega não verá automaticamente no dela.

Para compartilhar os dados entre a equipe, use o sistema de Exportar / Importar JSON.
2
Fluxo recomendado para equipes pequenas
Defina uma pessoa responsável pelos lançamentos (ex: secretaria).
Ela faz todos os lançamentos no sistema.
Periodicamente (semanal ou mensal), ela vai em ⚙️ Configurações → Exportar atual,
copia o JSON e envia pelo WhatsApp ou e-mail para as outras pessoas.
As outras pessoas vão em ⚙️ Configurações → Importar, colam o JSON e clicam Importar.
Pronto — todos estão com os mesmos dados.
📋O JSON é apenas texto — pode copiar e colar no WhatsApp, e-mail, Google Drive ou qualquer lugar.
3
Quando atualizar o site (subir novos arquivos)
Se um dia você precisar atualizar os arquivos do sistema (novas funcionalidades),
vá no Cloudflare Pages → seu projeto → Create new deployment → faça upload da pasta nova.
Os dados salvos nos navegadores não são apagados — eles ficam no localStorage, separado dos arquivos do site.
